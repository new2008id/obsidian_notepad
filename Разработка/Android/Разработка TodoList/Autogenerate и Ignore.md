#Autogenerate #Ignore #PrimaryKey #Room 
### Основные понятия

Как мы знаем, у `id` в `Note` добавили аннотацию `PrimaryKey()`, который отмечает эту переменную как первичный ключ и также автоматически генерирует его. Но в классе `AddNoteActivity` мы добавляем `id` через размер «БД», но настоящая база данных будет замешательстве, потому что он сам генерирует ключи, но мы вручную их добавляем и по этой причине БД откажется принимать ключ и строка не будет добавлена. 

>[!success ] Однако `PrimaryKey()` имеет такую возможность: если у `id` будет установлено `0`, то оно подумает, что ключ не установлен и он сам его сгенерирует.
### Crash из-за двух конструкторов

Если у вас есть класс, который выступает как таблица, то у него будет один конструктор, но возможно вы заходите добавить ещё конструктор, например, без `id`. `Room` не сможет понять, какой конструктор выбрать и он будет в замешательстве и пой это причине будет ошибка во время сборки.

>[!warning ] Для того, чтобы решить данную проблему — добавьте для нового конструктора аннотацию `@Ignore`. Мы явно скажем `Room`, чтобы он не рассматривал этот конструктор, а рассматривал только единственную.
### Примеры использования:

```java
public Note(int id, String text, int priority) {  
        this.id = id;  
        this.text = text;  
        this.priority = priority;  
    }  
  
    @Ignore  
    public Note(String text, int priority) {  
        this(0, text, priority);  
    }
```
